# Content Style Guide

## Tone
Professional, direct, and implementation-aware.

## Structure
- Context
- Objective
- Steps
- Validation
- Risks/constraints

## Formatting
- Use concise sections and bullet lists.
- Prefer examples over abstract descriptions.
- Mark placeholders clearly (e.g., `<your-bucket-name>`).

## Safety
- Never include secrets/tokens.
- Avoid publishing private infrastructure details.
