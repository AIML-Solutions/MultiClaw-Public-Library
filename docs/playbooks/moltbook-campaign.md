# Moltbook Campaign Playbook — MultiClaw

## Objective
Use Moltbook to publish a coherent narrative of MultiClaw progress that attracts practical opportunities (contracts, roles, collaborations).

## Content pillars
1. **Execution evidence**
   - Live repos
   - Live websites
   - CI quality gates
   - Meeting minutes and roadmap artifacts
2. **Technical edge**
   - Quant + derivatives data workflows
   - MCP-driven agentic tool architecture
   - Blockchain tracing and smart contract testing
3. **Commercial relevance**
   - Cost-aware engineering
   - Faster delivery cycles
   - Decision-grade dashboards

## Post sequence (suggested)
1. Launch post (company + architecture)
2. Quant stack post (LEAN + options/greeks + data quality)
3. MLFlow/LLM post (benchmarking + model routing)
4. Blockchain post (chain analysis + contract quality)
5. Frontend post (decision UX + docs viewer + execution board)

## Per-post structure
- Hook (problem + why it matters)
- Build snapshot (what was shipped)
- Evidence links (repo/site/docs)
- CTA (hire/contact/collaborate)

## Minimum evidence checklist before posting
- [ ] Target repo README updated
- [ ] CI badge passing
- [ ] One screenshot or chart
- [ ] One concrete metric or capability statement
- [ ] Link to roadmap/minutes artifact

## Notes
- Keep tone professional and direct.
- Avoid overclaiming.
- Emphasize reliability and execution depth.
