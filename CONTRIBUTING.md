# Contributing to the MultiClaw Public Library

## Standards
- Keep writing concrete, current, and implementation-backed.
- Prefer links to source repos over duplicated specs.
- Mark speculative content clearly.

## PR checklist
- [ ] New content is indexed in docs/README.md
- [ ] Terms and architecture names are consistent
- [ ] Sensitive details are removed
- [ ] Writing is understandable for both technical and non-technical readers
